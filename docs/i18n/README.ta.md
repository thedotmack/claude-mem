## விரைவு தொடக்கம்

ஒரே கட்டளையில் நிறுவுக:

```bash
npx claude-mem install
```

அல்லது Gemini CLI-க்கு நிறுவுக (தானாகவே `~/.gemini`-ஐ கண்டறியும்):

```bash
npx claude-mem install --ide gemini-cli
```

அல்லது OpenCode-க்கு நிறுவுக:

```bash
npx claude-mem install --ide opencode
```

அல்லது Claude Code-ல் உள்ள plugin சந்தையிலிருந்து நிறுவுக:

```bash
/plugin marketplace add thedotmack/claude-mem

/plugin install claude-mem
```

Claude Code அல்லது Gemini CLI-ஐ மறுதொடக்கம் செய்யுங்கள். முந்தைய அமர்வுகளின் சூழல் புதிய அமர்வுகளில் தானாகவே தோன்றும்.

> **குறிப்பு:** Claude-Mem npm-லும் வெளியிடப்பட்டுள்ளது, ஆனால் `npm install -g claude-mem` என்பது **SDK/நூலகத்தை மட்டுமே** நிறுவுகிறது — இது plugin hooks-ஐ பதிவு செய்யவோ அல்லது worker சேவையை அமைக்கவோ செய்யாது. எப்போதும் `npx claude-mem install` அல்லது மேலே உள்ள `/plugin` கட்டளைகள் மூலம் நிறுவுக.

### 🦞 OpenClaw Gateway

[OpenClaw](https://openclaw.ai) gateways-ல் claude-mem-ஐ நிலையான நினைவக plugin ஆக ஒரே கட்டளையில் நிறுவுக:

```bash
curl -fsSL https://install.cmem.ai/openclaw.sh | bash
```

நிறுவியானது சார்புகள், plugin அமைப்பு, AI வழங்குனர் கட்டமைப்பு, worker தொடக்கம், மற்றும் Telegram, Discord, Slack மற்றும் பலவற்றுக்கு விருப்பமான நேரடி கண்காணிப்பு ஊட்டங்களை கையாளுகிறது. விவரங்களுக்கு [OpenClaw Integration Guide](https://docs.claude-mem.ai/openclaw-integration)-ஐ பார்க்கவும்.

**முக்கிய அம்சங்கள்:**

- 🧠 **நிலையான நினைவகம்** - அமர்வுகள் முழுவதும் சூழல் நீடிக்கும்
- 📊 **படிப்படியான வெளிப்பாடு** - டோக்கன் செலவு தெரிவுடன் அடுக்கு நினைவக மீட்டெடுப்பு
- 🔍 **திறன்-அடிப்படையிலான தேடல்** - mem-search திறன் மூலம் உங்கள் திட்ட வரலாற்றை வினவுங்கள்
- 🖥️ **வலை பார்வையாளர் UI** - http://localhost:37777-ல் நேரடி நினைவக ஸ்ட்ரீம்
- 💻 **Claude Desktop திறன்** - Claude Desktop உரையாடல்களில் நினைவகத்தை தேடுங்கள்
- 🔒 **தனியுரிமை கட்டுப்பாடு** - சேமிப்பிலிருந்து முக்கியமான உள்ளடக்கத்தை விலக்க `<private>` குறிச்சொற்களைப் பயன்படுத்துங்கள்
- ⚙️ **சூழல் கட்டமைப்பு** - எந்த சூழல் செலுத்தப்படுகிறது என்பதில் நுணுக்கமான கட்டுப்பாடு
- 🤖 **தானியங்கி செயல்பாடு** - கைமுறை தலையீடு தேவையில்லை
- 🔗 **மேற்கோள்கள்** - ID-கள் மூலம் கடந்தகால கண்காணிப்புகளை குறிப்பிடுங்கள் (http://localhost:37777/api/observation/{id} மூலம் அணுகவும் அல்லது http://localhost:37777-ல் வலை பார்வையாளரில் அனைத்தையும் காணவும்)
- 🧪 **Beta சேனல்** - பதிப்பு மாற்றம் மூலம் Endless Mode போன்ற சோதனை அம்சங்களை முயற்சிக்கவும்

## இது எவ்வாறு செயல்படுகிறது

**மூல கூறுகள்:**

1. **5 வாழ்க்கைச்சுழல் Hooks** - SessionStart, UserPromptSubmit, PostToolUse, Stop, SessionEnd (6 hook ஸ்கிரிப்ட்கள்)
2. **சிறந்த நிறுவல்** - தற்சேமிக்கப்பட்ட சார்பு சரிபார்ப்பி (pre-hook ஸ்கிரிப்ட், வாழ்க்கைச்சுழல் hook அல்ல)
3. **Worker சேவை** - Bun மூலம் நிர்வகிக்கப்படும் வலை பார்வையாளர் UI மற்றும் 10 தேடல் முனைப்புகளுடன் 37777 போர்ட்டில் HTTP API
4. **SQLite தரவுத்தளம்** - அமர்வுகள், கண்காணிப்புகள், சுருக்கங்களை சேமிக்கிறது
5. **mem-search திறன்** - படிப்படியான வெளிப்பாட்டுடன் இயற்கை மொழி வினவல்கள்
6. **Chroma வெக்டார் தரவுத்தளம்** - புத்திசாலி சூழல் மீட்டெடுப்புக்கான கலப்பு சொற்பொருள் + முக்கிய வார்த்தை தேடல்

விவரங்களுக்கு [கட்டமைப்பு கண்ணோட்டம்](https://docs.claude-mem.ai/architecture/overview)-ஐ பார்க்கவும்.

## MCP தேடல் கருவிகள்

<!-- FIX 5: Added __IMPORTANT as the 4th tool so the stated count of 4 matches the listed tools -->
Claude-Mem, டோக்கன்-திறமையான **3-அடுக்கு பணிப்பாய்வு முறையை** பின்பற்றும் **4 MCP கருவிகள்** மூலம் புத்திசாலி நினைவக தேடலை வழங்குகிறது:

**3-அடுக்கு பணிப்பாய்வு:**

1. **`search`** - ID-களுடன் சுருக்கமான அட்டவணையை பெறுங்கள் (~50-100 டோக்கன்கள்/முடிவு)
2. **`timeline`** - சுவாரஸ்யமான முடிவுகளைச் சுற்றி காலவரிசை சூழலை பெறுங்கள்
3. **`get_observations`** - வடிகட்டப்பட்ட ID-களுக்கான முழு விவரங்களை மட்டுமே பெறுங்கள் (~500-1,000 டோக்கன்கள்/முடிவு)

**இது எவ்வாறு செயல்படுகிறது:**
- Claude MCP கருவிகளைப் பயன்படுத்தி உங்கள் நினைவகத்தை தேடுகிறது
- முடிவுகளின் அட்டவணையை பெற `search`-ஐ தொடங்குங்கள்
- குறிப்பிட்ட கண்காணிப்புகளைச் சுற்றி என்ன நடந்தது என்பதை காண `timeline`-ஐ பயன்படுத்துங்கள்
- தொடர்புடைய ID-களுக்கான முழு விவரங்களை பெற `get_observations`-ஐ பயன்படுத்துங்கள்
- விவரங்களை பெறுவதற்கு முன் வடிகட்டுவதன் மூலம் **~10x டோக்கன் சேமிப்பு**

**கிடைக்கும் MCP கருவிகள்:**

1. **`__IMPORTANT`** - மற்ற mem கருவிகளைப் பயன்படுத்துவதற்கு முன்பு எப்போதும் இதை அழைக்கவும்; சரியான பணிப்பாய்வுக்கான முக்கியமான வழிகாட்டுதல்களை வழங்குகிறது
2. **`search`** - வகை/தேதி/திட்டம் மூலம் வடிகட்டி, முழு-உரை வினவல்களுடன் நினைவக அட்டவணையை தேடுங்கள்
3. **`timeline`** - குறிப்பிட்ட கண்காணிப்பு அல்லது வினவலைச் சுற்றி காலவரிசை சூழலை பெறுங்கள்
4. **`get_observations`** - ID-களால் முழு கண்காணிப்பு விவரங்களை பெறுங்கள் (எப்போதும் பல ID-களை தொகுதியாக பெறுங்கள்)

**எடுத்துக்காட்டு பயன்பாடு:**

```typescript
// படி 1: அட்டவணையை தேடுங்கள்
search(query="authentication bug", type="bugfix", limit=10)

// படி 2: அட்டவணையை மதிப்பாய்வு செய்யுங்கள், தொடர்புடைய ID-களை அடையாளம் காணுங்கள் (எ.கா., #123, #456)

// படி 3: முழு விவரங்களை பெறுங்கள்
get_observations(ids=[123, 456])
```

விரிவான எடுத்துக்காட்டுகளுக்கு [தேடல் கருவிகள் வழிகாட்டி](https://docs.claude-mem.ai/usage/search-tools)-ஐ பார்க்கவும்.

## அமைவுகள்

அமைவுகள் `~/.claude-mem/settings.json`-ல் நிர்வகிக்கப்படுகின்றன (முதல் இயக்கத்தில் இயல்புநிலைகளுடன் தானாகவே உருவாக்கப்படும்). AI மாதிரி, worker போர்ட், தரவு அடைவு, பதிவு அளவு, மற்றும் சூழல் செலுத்தல் அமைவுகளை கட்டமைக்கவும்.

அனைத்து கிடைக்கக்கூடிய அமைவுகள் மற்றும் எடுத்துக்காட்டுகளுக்கு **[அமைவுகள் வழிகாட்டி](https://docs.claude-mem.ai/configuration)**-ஐ பார்க்கவும்.

### பயன்முறை & மொழி கட்டமைப்பு

Claude-Mem `CLAUDE_MEM_MODE` அமைவு மூலம் பல பணிப்பாய்வு பயன்முறைகள் மற்றும் மொழிகளை ஆதரிக்கிறது.

இந்த விருப்பம் இரண்டையும் கட்டுப்படுத்துகிறது:
- பணிப்பாய்வு நடத்தை (எ.கா. code, chill, investigation)
- உருவாக்கப்பட்ட கண்காணிப்புகளில் பயன்படுத்தப்படும் மொழி

#### எவ்வாறு கட்டமைப்பது

`~/.claude-mem/settings.json`-ல் உங்கள் அமைவு கோப்பை திருத்துங்கள்:

<!-- FIX 3: Removed unsupported code--ta example; code--ta.json does not exist in the repo.
     Use an existing supported mode as the example instead. -->
```json
{
  "CLAUDE_MEM_MODE": "code--zh"
}
```

பயன்முறைகள் `plugin/modes/`-ல் வரையறுக்கப்பட்டுள்ளன. உள்நாட்டில் அனைத்து கிடைக்கக்கூடிய பயன்முறைகளையும் காண:

```bash
ls ~/.claude/plugins/marketplaces/thedotmack/plugin/modes/
```

#### கிடைக்கக்கூடிய பயன்முறைகள்

| பயன்முறை | விளக்கம் |
|------------|-------------------------|
| `code` | இயல்புநிலை ஆங்கில பயன்முறை |
| `code--zh` | எளிமையான சீன பயன்முறை |
| `code--ja` | ஜப்பானிய பயன்முறை |

மொழி-குறிப்பிட்ட பயன்முறைகள் `code--[lang]` முறையை பின்பற்றுகின்றன, இங்கு `[lang]` என்பது ISO 639-1 மொழி குறியீடு (எ.கா., `zh` சீனத்திற்கு, `ja` ஜப்பானியத்திற்கு).

#### பயன்முறையை மாற்றிய பிறகு

புதிய பயன்முறை கட்டமைப்பை பயன்படுத்த Claude Code-ஐ மறுதொடக்கம் செய்யுங்கள்.
